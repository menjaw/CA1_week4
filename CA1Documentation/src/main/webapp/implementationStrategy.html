The Implementation Strategy

Domain model

Inheritance Strategy: Table per class

  • Add the entity annotations to the class

  • Add the mapping annotations to the attributes

  • Each class get mapped to its own database table

  • The mapping allows you to use polymorphic queries and define relationships to the superclass

  • This strategy is simular to the 'Mapped Superclasses Strategy' but here the superclass is also an Entity-class

Benefits:

  • Easy and efficient to make queries for a specific entity-class

Drawbacks:

  • This table structure adds a lot of complexity to polymorphics queries

Why this Strategy

We thought this strategy would be easy to work with and would keep it as simple as possible

But we ran in to a lot of challanges and therefor we couldn't finish the application
